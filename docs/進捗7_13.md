# OCRデモページ作成PJ\_作業進捗報告\_2026_7_13

## 1. 目的

独立OCRをローカル環境とDocker Container内で実行し、画像から正規化済みJSONを取得できることを確認する。

また、既存OCRのDjango依存部分とFastAPIの`UploadFile`形式の差をAdapterで吸収し、Browser確認へ進める状態にする。

## 2. 結論

`python3 run_ocr.py sample.png`を実行し、ローカルTerminalでOCR結果のJSON表示を確認した。

DockerfileからImageを作成し、Containerを起動した後、Container内でも同じOCR処理が成功した。

FastAPIの`UploadFile`と既存OCRが要求する同期ファイル形式の差については、`SyncImageFile` Adapterを追加して対応した。

## 3. 実施内容

### 3.1 独立OCRの単体実行

```text
対象：
run_ocr.py、sample.png、ocr_core

実施内容：
python3 run_ocr.py sample.pngを実行し、
OCR結果がJSONとしてTerminalへ表示されることを確認した。

理由：
FastAPIやDockerへ接続する前に、
OCR本体が単独で正常動作することを確認するため。
```

### 3.2 FastAPI用Adapterの追加

```text
対象：
app.py、SyncImageFile

実施内容：
FastAPIのUploadFileをawait img.read()でbytesへ変換し、
filenameとbytesをSyncImageFileへ渡す処理を追加した。

理由：
既存OCRは同期的なread()を持つファイル形式を前提としており、
FastAPIのUploadFileをそのまま渡せなかったため。
```

処理経路：

```text
UploadFile
↓
await img.read()
↓
bytes
↓
SyncImageFile
↓
既存OCR
```

### 3.3 Docker環境でのOCR実行

```text
対象：
Dockerfile、Docker Image、Docker Container

実施内容：
DockerfileからImageを作成し、
ImageからContainerを作成・起動した。

Container内でsample.pngをOCR処理し、
JSONがTerminalへ表示されることを確認した。

理由：
ローカルMac環境だけでなく、
再現可能なDocker環境でもOCRが動作することを確認するため。
```

### 成功結果

・Terminalへ正規化済みJSONが表示された

### 問題・対応

```text
問題：
FastAPIのUploadFileを既存OCRへ直接渡せなかった。

原因：
UploadFile.read()は非同期処理だが、
既存OCRは同期的なread()を持つ形式を想定していた。

対応：
await img.read()でbytesを取得し、
SyncImageFileへ格納して既存OCRへ渡した。

結果：
既存OCRを大きく変更せず、FastAPIから利用できる構成になった。
```
