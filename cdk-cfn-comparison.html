🔑 CDK → CFn 変換ルールまとめ

1. 論理IDの決定

CDKコンストラクタの第2引数がCloudFormationの論理IDになる。
new CfnContainer(this, 'OcrDemoContainerService', ...)
→ 論理ID: OcrDemoContainerService

2. Parameter参照 → Ref

param.valueAsString / param.valueAsNumber
→ {"Ref": "パラメータ名"} に変換

3. 属性アクセス → GetAtt

resource.attrXxx
→ {"Fn::GetAtt": ["論理ID", "Xxx"]} に変換

4. プロパティ名のケース変換

CDK (camelCase) → CFn (PascalCase)
serviceName → ServiceName
containerServiceDeployment → ContainerServiceDeployment

5. 静的値のインライン展開

CDKで固定文字列を設定した場合はそのまま埋め込まれる
serviceName: 'ocr-demo-...' → "ServiceName": "ocr-demo-..."

6. メタデータ自動追加

CDKはブートストラップ情報・メタデータ・Conditionsを自動で付与。
開発者が明示的に書く必要はない。
